Beauty AI Assistant

Beauty AI Assistant — цифровой сервис персонального подбора косметического ухода для клиентов сети «Золотое яблоко». Сервис помогает выбрать базовый план ухода на основе параметров кожи и предпочтений.

Что доступно в сервисе

  • Описание и логика сервиса (раздел «Описание»);
  • Интерактивный сценарий подбора ухода (раздел «Ассистент»);
  • Аналитика, цели и отчётность (раздел «Аналитика»);
  • Формализованные требования к сервису (раздел «Техническое задание»).

Перейдите в раздел «Ассистент», чтобы получить пример рекомендаций.

Веб-аналитика и отчётность

1. Сбор данных

  • Яндекс.Метрика: визиты, источники, глубина просмотра, вебвизор, карты кликов и скроллинга.
  • GA4 (при необходимости): воронки, анализ каналов, события и аудитории.
  • Качественные данные: формы обратной связи и опросы для понимания ожиданий пользователей.

2. Техническая подготовка

  • логичная структура разделов (Описание / Ассистент / Аналитика / ТЗ);
  • отслеживание кликов и переходов;
  • читаемый интерфейс и доступность (контраст, крупные элементы).

3. Цели и события

  • login_success — успешный вход в систему;
  • tab_1 / tab_2 / tab_3 / tab_4 — переходы между разделами;
  • recommendation_click — запрос подбора ухода;
  • logout — выход из аккаунта.

4. Что анализируется

  • источники трафика и динамика визитов;
  • поведенческие метрики: время, глубина, карта кликов;
  • конверсия: вход → ассистент → «Подобрать уход»;
  • поиск точек улучшения интерфейса по данным вебвизора.

5. Отчётность

На основании событий формируются отчёты по конверсиям и пользовательским сценариям. При необходимости данные экспортируются в BI для дашбордов.

Техническое задание

Документ определяет требования к веб-сервису Beauty AI Assistant: функциональность, ограничения, требования к интерфейсу и аналитике, а также критерии приёмки.

Проект

Название: Beauty AI Assistant

Назначение: персональный подбор ухода за кожей в экосистеме сети «Золотое яблоко»

Формат: web-приложение (одностраничный интерфейс с разделами)

Цели
  • упростить подбор базового ухода по параметрам кожи;
  • повысить удобство и скорость выбора продуктов;
  • собрать данные о пользовательских сценариях для улучшения сервиса.

1. Область применения

Сервис предназначен для клиентов сети «Золотое яблоко» и может быть использован как в веб-версии, так и как часть будущей интеграции в мобильное приложение/личный кабинет.

2. Пользователи и роли

  • Пользователь — получает рекомендации по уходу и взаимодействует с интерфейсом.
  • Администратор/аналитик — анализирует метрики, конверсии и события (через Метрику/GA4).

3. Функциональные требования

3.1 Авторизация

  • страница входа содержит поля e-mail и пароль;
  • после успешного входа экран авторизации скрывается, доступ открывается к разделам;
  • выход из аккаунта возвращает пользователя на страницу входа.

3.2 Раздел «Описание»

  • описание назначения сервиса и пользовательского сценария;
  • краткая навигационная подсказка по разделам.

3.3 Раздел «Ассистент»

  • форма анкеты (тип кожи, запросы, бюджет, предпочтения);
  • кнопка «Подобрать уход» формирует и отображает список рекомендаций;
  • интерфейс должен быть читаемым и понятным (крупные элементы, контраст).

3.4 Раздел «Аналитика»

  • описание инструментов аналитики и целей;
  • перечень метрик и подходов к отчётности.

3.5 Раздел «Техническое задание»

  • формализованные требования к сервису;
  • критерии приёмки и ограничения.

4. Нефункциональные требования

Интерфейс
  • адаптивность (ПК/планшет/телефон);
  • высокая читаемость (контраст, размер шрифтов);
  • навигация фиксирована и доступна на всех экранах.
Производительность
  • быстрая загрузка (статический сайт на GitHub Pages);
  • минимум внешних зависимостей;
  • корректная работа в современных браузерах.

5. Аналитика: события и цели

  • login_success — успешный вход;
  • tab_1 / tab_2 / tab_3 / tab_4 — переход между разделами;
  • recommendation_click — нажатие «Подобрать уход»;
  • logout — выход из аккаунта.

6. Безопасность и защита данных

6.1 Доступ и авторизация

  • Доступ к разделам «Описание», «Ассистент», «Аналитика», «Техническое задание» предоставляется только после входа в систему.
  • При выходе из аккаунта доступ к разделам закрывается, пользователь возвращается на экран авторизации.
  • Проверка полей e-mail и пароля обязательна (пустые значения не допускаются).

6.2 Обработка персональных данных

  • Сервис не запрашивает и не хранит лишние персональные данные (кроме e-mail для демонстрации входа).
  • Пароль не сохраняется и не передаётся третьим лицам; используется только для проверки на стороне интерфейса.
  • Рекомендации формируются без сохранения введённых данных на сервере (в пределах текущей сессии браузера).

6.3 Безопасность публикации и передачи данных

  • Размещение сайта осуществляется на GitHub Pages с доступом по HTTPS.
  • Для сторонних скриптов (Яндекс.Метрика) используются официальные источники подключения.
  • Запрещено добавлять в код приватные ключи, пароли и токены (если в будущем появятся интеграции).

6.4 Аналитика и конфиденциальность

  • Яндекс.Метрика фиксирует только события и поведение на сайте (клики/переходы), без передачи паролей.
  • При использовании Вебвизора содержимое полей пароля не должно записываться (поле типа password скрывается автоматически).
  • При необходимости на сайте может быть добавлено уведомление о сборе аналитики (cookies/метрики).

7. Критерии приёмки

  • вход/выход работают корректно, экран авторизации появляется только при выходе;
  • разделы доступны только после входа;
  • нажатие «Подобрать уход» отображает рекомендации и фиксируется в Метрике как цель;
  • переходы по разделам фиксируются как цели tab_1…tab_4;
  • интерфейс читабелен и стабилен при прокрутке.
